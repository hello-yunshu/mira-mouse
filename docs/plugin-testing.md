# Plugin Testing

Every read and write workflow needs success, timeout, malformed response, oversized response, unplug, status mismatch, and unknown-field preservation Fixtures. Passing them is `fixture-verified`, not hardware evidence. Hardware records must identify model, VID/PID, firmware, connection, operation, readback, verifier, date, and result.
