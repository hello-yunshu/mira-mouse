# 路线图

本文档描述 Mira 的发展方向与阶段性目标。版本号遵循项目规则（不含 `4` 与 `11`），具体发布时间以实际进度为准。

## 当前状态

**版本 0.9.15** —— 已实现三端发布、双协议支持、声明式插件架构与本地 AI 续航预测。

| 领域 | 状态 |
|---|---|
| macOS / Windows / Linux 安装包 | 已发布 |
| AMaster（Protocol A）支持 | 已支持 |
| Logitech（HID++ 2.0）支持 | 已支持 |
| 声明式插件包与签名验证 | 已实现 |
| 本地 AI 续航预测（9 维特征） | 已集成 |
| 未签名社区发布链 | 已建立 |

## 近期目标

**完善插件生态与发布安全**

- 扩展协议 DSL：支持 checksum、field、fragmentation、read-modify-write 与有限状态机操作，保持有类型、有版本、有界且可 Fixture 测试。
- 建立不可变的签名插件发布链，确保插件包从签名到分发的完整性可验证。
- 发布经评审的只读 AMaster 插件候选版本，附带完整的 Fixture 证据。
- 完善 release CI 的 SBOM 与第三方清单生成流程，使依赖来源可溯源。

## 中期目标

**扩展设备覆盖与连接方式**

- 增加更多品牌与型号的插件支持，通过社区贡献扩展设备生态。
- 评估蓝牙连接的可行性，在确保安全边界的前提下扩展传输方式。
- 丰富本地 AI 能力：基于更多设备上下文维度的续航预测与使用习惯分析。
- 完善多设备并发管理体验，优化设备热插拔与状态恢复。

## 长期方向

**构建可持续的社区驱动生态**

- 建立插件市场与分发机制，使第三方插件可在安全边界内被发现与安装。
- 支持配置导入导出与跨设备配置迁移。
- 探索更多本地化能力（如使用模式统计、设备健康度评估），所有数据保持在本地。
- 在保持无遥测、无账号承诺的前提下，为社区提供可验证的安全审计材料。

## 不在计划内

以下能力明确不在 Mira 的路线图中：

- **遥测与数据上报** —— 永远不会加入任何形式的用户数据收集。
- **账号体系** —— 不会要求用户注册或登录。
- **厂商授权** —— 不会寻求设备厂商的官方授权或背书。
- **可执行插件代码** —— 插件始终保持声明式，不会引入脚本或代码执行能力。
- **任意网络访问** —— 仅保留更新检查所需的网络请求，不会扩展为常驻网络服务。
