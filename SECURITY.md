# Security Policy

No version is supported yet. Use GitHub Private Vulnerability Reporting after the repository owner enables it; do not open a public issue containing an exploit, serial number, signing key, or unredacted diagnostic. A plugin/update key leak requires immediate key revocation, index removal, affected release withdrawal, a new key ID, and user-facing incident guidance. Repository settings that require web access remain listed as `blocked` in `docs/repository-settings.md`.
